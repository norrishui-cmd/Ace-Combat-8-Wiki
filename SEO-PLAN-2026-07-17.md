# Ace Combat 8 Wiki SEO 计划（2026-07-17）

## 当前判断

网站已具备 Astro SSG、canonical、sitemap、robots.txt、基础结构化数据、面包屑和内容集合，技术底座可继续扩展。第一阶段主要问题不是“页面太少”本身，而是现有 URL 过度集中于发售前新闻和宽泛介绍，缺少能准确承接玩家搜索问题的独立答案页；部分页面还存在事实状态滞后和标题过长的风险。

本轮采取“主题集群 + 质量门”策略，不批量生成空飞机、任务或角色页。所有新页面必须有：唯一意图、开头直接答案、官方证据边界、独立内容、FAQ、面包屑、相邻内链、canonical、sitemap lastmod，并通过构建和自动审计。

## 第一轮已执行

### 1. 平台兼容性集群

- `/platforms`：平台总入口和状态矩阵
- `/platforms/ps5`
- `/platforms/xbox-series-x-s`
- `/platforms/pc`
- `/platforms/ps4`
- `/platforms/xbox-one`
- `/platforms/nintendo-switch-2`
- `/platforms/steam-deck`

这些页面分别承接购买前的明确搜索意图。未公布平台统一写明“截至 2026-07-17 尚未公布”，不把推测写成永久否定。

### 2. 商业与问答入口

- `/ace-pass`：区分免费、Premium、Premium Plus、30 级跳过和 Morgan–PIXY Season 1 奖励，避免与 Editions 页重复介绍整套版本。
- `/faq`：聚合 release、platform、purchase、online、gameplay 问题，把链接权重导向具体答案页。

### 3. 技术与事实修复

- 标题统一缩短站点后缀，降低 SERP 截断风险。
- 默认 robots 增加 `index, follow, max-image-preview:large`，404 继续 noindex。
- VideoGame 结构化数据补充准确的平台、类型和官方 `sameAs`。
- 导航、首页和页脚增加 Platforms、FAQ、Ace Pass 入口。
- sitemap 为新 URL 配置真实 lastmod 与分层优先级。
- 将 VR 从无充分依据的“Not supported”修正为“Not announced”。
- PC 配置页补充 Steam 官方最低配置表，并明确推荐配置仍为 TBD。
- 新增 `npm run audit:seo`：检查 title、description、canonical、H1、重复元数据、低字数和断链。

## 30 天执行节奏

1. 每周核对 Bandai Namco、Steam、PlayStation 和 Xbox 官方页面，更新日期敏感结论。
2. 每次发布 5–12 个有完整答案的 URL；若内容证据不足，宁可暂停，不设凑数目标。
3. 上线后在 GSC 按 query/page 对比 7 天与 28 天，重点观察 `ace combat 8 + platform`、`system requirements`、`ace pass`、`preorder` 集群。
4. 新 URL 先从 `/platforms`、`/faq`、首页和相关正文获得内链，再提交 sitemap；优先检查 P0 URL 的抓取与 canonical。
5. 标题/摘要修改后至少观察 5–7 天；优先修复有曝光但 CTR 低的页面，不连续改动同一批标题。

## 质量门

- 必须在首屏回答查询，不能只有“等待更新”。
- 一个 URL 一个主意图，避免 release/edition/preorder 互相复制。
- 所有状态用日期限定，并链接官方一手来源。
- 不虚构飞机数值、任务、奖励、模式、平台或性能结果。
- 构建、SEO 审计、内部链接和代表性页面人工检查全部通过后才部署。
